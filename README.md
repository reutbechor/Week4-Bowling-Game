Weekly Assignment – Game Development Course

# קישור לאתר itch -


## תיאור המשחק  
הפרויקט הוא משחק באולינג דו־ממדי (2D) , בצורה מעט שונה (התחלנו מבאולינג והפכנו את זה למשחק יותר חוויתי לדעתנו).  
השחקן גורר את הכדור עם העכבר כדי לקבוע את עוצמת וכיוון הזריקה, השחקן יכול לכוון לכל מקום.  
לשחקן יש שתי זריקות בלבד:

- אם לאחר שתיהן כל 10 הפינים נפלו - ניצחון ומעבר לשלב הבא 
- אם נשארו פינים עומדים - הפסד  

המשחק כולל פיזיקה מלאה, מחיקת פינים שנפלו, מערכת תורות, ניקוד, טקסטי תוצאה ומעבר בין שלבים.

---

# מבנה וארכיטקטורת הקוד

הפרויקט בנוי משלושה סקריפטים עיקריים:

---

## 1. BallController.cs  
אחראי על שליטה בכדור.

**אחריות:**
- זיהוי גרירת עכבר (OnMouseDown / OnMouseUp)
- חישוב וקטור הזריקה
- שימוש ב-Rigidbody2D להפעלת כוח
- איפוס הכדור בין תורות
- קריאה ל־PinsManager כאשר הכדור נזרק:
```
pinsManager.OnBallThrown();
```

---

## 2. Pin.cs  
מייצג פין יחיד במסלול.

**אחריות:**
- שמירת מיקום וסיבוב התחלתיים  
- בדיקת נפילה: אם זווית הסיבוב גדולה מ־20 מעלות  
- איפוס פין (רק לתור השני)  
- מאפשר ל־PinsManager למחוק אותו מהמשחק כשנפל  

---

## 3. PinsManager.cs  
חלק הלוגיקה הראשי של המשחק.

**אחריות:**
- איסוף כל הפינים בתחילת השלב
- קבלת אירוע זריקה מהכדור
- המתנה לסיום הפיזיקה (2.5 שניות)
- בדיקה כמה פינים נפלו
- מחיקת פינים שנפלו
- שמירת ספירה מצטברת
- מערכת שני התורות:
  - תור ראשון - איפוס מי ששרד  
  - תור שני - סוף משחק
- הצגת תוצאה על המסך
- טעינת שלב חדש או התחלה מחדש

# תרשים מחלקות עיקריות 

```
BallController
 ├─ Rigidbody2D rb
 ├─ Vector3 startPos
 ├─ PinsManager pinsManager
 ├─ OnMouseDown()
 ├─ OnMouseUp()
 └─ ResetBall()
        │
        ▼
PinsManager
 ├─ List<Pin> pins
 ├─ int turn
 ├─ int totalKnocked
 ├─ OnBallThrown()
 ├─ WaitAndCheckPins()
 ├─ ResetStandingPins()
 ├─ EndGame()
 ├─ LoadNextLevel()
 └─ RestartLevel()
        │
        ▼
Pin
 ├─ Vector3 startPos
 ├─ Quaternion startRot
 ├─ IsKnockedDown()
 └─ ResetPin()
```

# לוגיקת המשחק – הסבר מלא

## שלבי הזריקה  
1. השחקן לוחץ על הכדור - נשמרת נקודת התחלה  
2. משחרר - מחושבת משיכת גרירה  
3. מופעל כוח על Rigidbody2D  
4. מחכים שהפיזיקה תסתיים  
5. בודקים פינים שנפלו  

ככל שהגרירה ארוכה יותר — הכוח חזק יותר.

---

# מערכת שני התורות

### תור ראשון:
- מחשבים כמה פינים נפלו  
- מוחקים את אלה שנפלו  
- הפינים שנותרו עומדים מאופסים בחזרה לנקודת ההתחלה  

### תור שני:
- הפלה נוספת של מה שנשאר  

### סוף:
- אם רשימת הפינים ריקה - ניצחון  
- אחרת - הפסד  

---

# מערכת טקסטים (UI)
הפרויקט משתמש ב-TextMeshPro ומציג:

- מספר הפינים שנפלו (scoreText)
- תוצאה סופית (resultText)

---

# תמיכה בשלבים (Levels)

המשחק תומך במעבר בין שלבים:  
במקרה של ניצחון נטען את הסצנה הבאה:

```
SceneManager.LoadScene(currentIndex + 1)
```

במקרה של הפסד נטען מחדש את השלב הנוכחי:

```
SceneManager.LoadScene(currentIndex)
```

אם הגענו לסצנה האחרונה — חוזר ל-Level1.
